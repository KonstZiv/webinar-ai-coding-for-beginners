# Послідовність промптів для створення проєкту

Ця таблиця описує 4 кроки для створення консольної гри Tic-Tac-Toe з AI-кодуванням.

| # | Промпт | Результат |
|---|--------|-----------|
| 1 | **Ініціалізуй Python-проєкт для консольної гри Tic-Tac-Toe. Використовуй `uv` для залежностей, додай `tic-tac-toe-3x3` та `rich`. Створи базову структуру `src/my_tic_tac_toe/`.** | `pyproject.toml`, `src/my_tic_tac_toe/__init__.py`, `.venv`, встановлені залежності |
| 2 | **Створи `RichRenderer` що наслідує `tic_tac_toe_3x3.game.renderers.Renderer`. Клітинки 17×7 символів, X (червоний) та O (синій) малюються блоками █, сітка з подвійних ліній (╔╗╚╝), колонки A/B/C, ряди 1/2/3, статус гри знизу, виграшні клітинки підсвічуються reverse style. Додай `__main__.py` для запуску.** | `renderer.py` з повною реалізацією, `__main__.py`, працююча гра `uv run python -m my_tic_tac_toe` |
| 3 | **Напиши pytest тести для RichRenderer: наслідування, розміри ASCII art, статуси гри, рендеринг клітинок, всі 8 виграшних комбінацій. Додай GitHub Actions CI з ruff lint та pytest.** | `tests/test_renderer.py` (40+ тестів), `.github/workflows/ci.yml` |
| 4 | **Додай CLI опції `--ai` (minimax/random) та `--ai-first` для гри з комп'ютером. Онови README.md з інструкціями запуску.** | Оновлений `__main__.py` з argparse, README.md з документацією |
